## UX Principles for Phase 1

### 1. Clarity first
User must understand the site in a few seconds.

### 2. No overload
Do not put too many cards, links, or categories on the first version.

### 3. Textual + visual balance
Pages should combine short text blocks with visual entry points.

### 4. Easy mobile reading
The structure must stay clean on iPhone screens.

### 5. Visible next step
Every page should suggest a next action.

## Navigation Rules
- Keep main nav short
- Avoid deep nesting in phase 1
- Use simple labels
- Keep CTA visible

━━━━━━━━━━━━━━━━━━━━

## CTA Rules
Primary CTA should appear:
- in hero
- near bottom of home page
- on resources/tutorial pages

CTA should lead to:
- starter pack
- join page
- selected key resource

━━━━━━━━━━━━━━━━━━━━

## Mobile Rules
- Short headlines
- Medium paragraph length
- Clean card layout
- Avoid crowded visual grids
